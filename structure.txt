========================================
ARCHITECTURE DE SÉCURITÉ
========================================

1. PROTECTION XSS (Cross-Site Scripting)
   - Échappement automatique Jinja2
   - Fonction sanitizeInput() dans script.js
   - Bibliothèque Bleach pour nettoyage HTML

2. PROTECTION SQL INJECTION
   - Utilisation exclusive de SQLAlchemy ORM
   - Requêtes paramétrées automatiques
   - Pas de concaténation SQL brute

3. PROTECTION CSRF (Cross-Site Request Forgery)
   - Tokens CSRF sur tous les formulaires
   - Validation côté serveur systématique
   - Flask-WTF pour gestion automatique

4. AUTHENTIFICATION SÉCURISÉE
   - Mots de passe hashés avec PBKDF2-SHA256
   - Sels cryptographiques de 16 caractères
   - Rate limiting sur les tentatives de connexion
   - Sessions sécurisées (HttpOnly, Secure, SameSite)

5. HEADERS DE SÉCURITÉ
   - Content-Security-Policy (CSP)
   - X-Frame-Options (protection clickjacking)
   - X-Content-Type-Options
   - Strict-Transport-Security (HSTS)
   - Via Flask-Talisman

6. VALIDATION DES DONNÉES
   - Validation côté client (JavaScript)
   - Validation côté serveur (Python)
   - Regex patterns sur les inputs
   - Types de données stricts

========================================
BASE DE DONNÉES MYSQL (Aiven Cloud)
========================================

Tables :

  users
    id, username, email, password_hash,
    is_admin, role, is_active, created_at, last_login

  personnels
    id, matricule, nom, prenom, fonction, societe,
    salaire, type_salaire, active, created_at

  projets
    id, nom, region, gouvernorat, ville, adresse,
    coordinates, date_debut_estimee, date_fin_estimee,
    etat  ← NOUVEAU (en_cours | planifie | termine, défaut en_cours)
    active, created_at

  deplacements
    id, personnel_id, projet_id,
    date_debut, heure_debut, date_fin, heure_fin,
    statut, created_at, created_by

  deplacement_validations
    id, deplacement_id, validated_by,
    action, commentaire, created_at

  work_schedules
    id, nom, heure_debut, heure_fin,
    is_active, created_at

  heures_supplementaires
    id, deplacement_id, heures,
    commentaire, created_by, created_at

  payroll_config                    ← NOUVEAU
    id, jour_debut (1-28),
    updated_by, updated_at

Relations :
  personnel     1:N  deplacements
  projet        1:N  deplacements
  user          1:N  deplacements         (created_by)
  deplacement   1:1  heures_supplementaires
  deplacement   1:N  deplacement_validations
  user          1:N  deplacement_validations (validated_by)
  user          0:1  payroll_config          (updated_by)


========================================
MIGRATIONS DB (db_tool.py)
========================================

Après toute mise à jour du projet, lancez :

  python db_tool.py check    → vérifie l'état de la DB
  python db_tool.py migrate  → applique les modifications manquantes
  python db_tool.py inspect  → affiche la structure complète

Migrations v2 (release actuelle) :
  • ALTER TABLE projets ADD COLUMN etat ENUM(...)
  • CREATE TABLE payroll_config

Les migrations sont idempotentes : relancer migrate plusieurs fois
ne crée pas de doublons et ne modifie pas les données existantes.


========================================
ROUTES PRINCIPALES
========================================

  GET  /                             → landing
  GET/POST /login                    → authentification
  GET  /logout                       → déconnexion

  GET  /dashboard                    → tableau de bord
  GET  /api/dashboard/fonctions      → stats par poste (AJAX)

  GET  /personnels                   → liste personnels
  POST /personnels/add               → ajouter
  POST /personnels/edit/<id>         → modifier
  POST /personnels/delete/<id>       → archiver (active=False)
  POST /personnels/toggle-active/<id>→ basculer actif/inactif (JSON)

  GET  /projets                      → liste projets
  POST /projets/add                  → ajouter (avec etat)
  POST /projets/edit/<id>            → modifier (avec etat)
  POST /projets/delete/<id>          → archiver
  GET  /api/projet/<id>              → données JSON (avec etat)
  GET  /api/projet/<id>/stats        → stats déplacements (déduplication)

  GET  /deplacements                 → liste déplacements
  POST /deplacements/add             → ajouter multi-personnel

  GET  /heures-supplementaires       → liste HS (filtres date/perso/projet/fonction)
  GET  /heures-supplementaires/export→ export XLSX (mêmes filtres)
  POST /heures-supplementaires/add   → ajouter/modifier HS
  POST /heures-supplementaires/delete/<id>

  GET/POST /heures-supplementaires/payroll-config → config mois salarial + séances
  GET  /api/hs/stats                 → KPIs HS (semaine + mois salarial configuré)

  GET  /validations                  → déplacements en attente
  GET  /users                        → gestion utilisateurs (admin)
  GET/POST /profile                  → profil utilisateur


========================================
INSTALLATION RAPIDE
========================================

1. Configurer .env (DB_USER, DB_PASSWORD, DB_HOST, DB_PORT, DB_NAME, SECRET_KEY)
2. pip install -r requirements.txt
3. python app.py         (crée les tables de base via SQLAlchemy)
4. python db_tool.py migrate  (ajoute les colonnes/tables supplémentaires)
5. Accès : http://localhost:5000
   Login par défaut : admin / Admin.Init!2026  (à changer immédiatement)
